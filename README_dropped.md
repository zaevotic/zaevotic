<h2 style="text-align: center; color: #ff6b6b;">That's Me</h2>
<div>
<div>
<p style="text-align: center;">I like working across layers: systems, networking, backend logic, and UI. Not really into “learning” tech, I’d rather take things apart, figure out what’s actually going on, and rebuild them into something that works. Using things is easy, understanding them is the point.</p>

<p style="text-align: center;">If it breaks, good. Now I <em>have</em> to understand it. That’s usually where things start getting interesting.</p>

<hr style="height: 1px; background-color: #30363d;"/>

<p>Most of what I build comes from curiosity:<br>
"how does this actually work?"<br>
→ break it<br>
→ debug it<br>
→ turn it into something usable</p>

<p> "Lately, that's looked like:</p>
<ul>
    <li>OS-level experiments (xv6, qemu, custom shells, architectures)</li>
    <li>Linux-heavy workflows (containers, systemd, remote setups)</li>
    <li>desktop + web apps (music player, system tools)</li>
    <li>AI/NLP systems like <em>nomi</em></li>
</ul>

<p>I'm most comfortable close to the system — terminals, processes, and things that aren't abstracted away too much.</p>
